Voici le script unique complet qui fait les 4 étapes :
crée les Content Views simples avec le même nom et le même label
ajoute les repositories aux CV en recherchant les repositories par name
crée les Composite Content Views avec le même nom et le même label
ajoute les Content Views composants dans les CCV
Il s’appuie sur tes fichiers export :
content_views_full.csv
content_view_repositories.csv
ccv_components.csv
Ce que fait ce script
Il conserve pour chaque CV et CCV :
le même name
le même label
et pour les repositories :
il fait le matching par repository name
il n’utilise pas le label du repository

Point d’attention important
Ton CSV est en vrai CSV avec guillemets, mais comme certains champs peuvent contenir des virgules, le parsing bash avec IFS=',' read ... peut devenir fragile si un champ description contient des virgules.
Dans ton export actuel, ça semble simple et ça passera probablement.
Pour un usage plus robuste en prod, la meilleure version est en Python ou avec un vrai parseur CSV.

1 CV simple
