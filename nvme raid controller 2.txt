Analyse de l’architecture NVMe et besoin éventuel d’un contrôleur RAID NVMe
Bonjour à tous,
Suite à la configuration récente du stockage et aux vérifications effectuées sur le serveur (Dell PowerEdge R760), nous avons analysé l’architecture actuelle des disques à partir du système d’exploitation ainsi que via l’interface iDRAC.
Nous avons identifié que le serveur dispose actuellement des contrôleurs suivants :
Contrôleur BOSS-N1 utilisé pour les disques du système (OS)
Contrôleur PERC H755 utilisé pour les disques SAS/SATA
Cependant, les disques NVMe ne sont pas connectés au contrôleur RAID. Ils sont directement connectés au CPU via PCIe. Cela signifie que le RAID matériel pour ces disques NVMe n’est pas pris en charge dans la configuration actuelle.
Par conséquent, la configuration actuelle des disques de données repose sur une solution de mirroring logiciel utilisant LVM RAID1 afin d’assurer la redondance et la protection des données.
Bien que cette configuration soit fonctionnelle et couramment utilisée dans les environnements Linux modernes, si nous souhaitons gérer les disques NVMe via un RAID matériel (gestion centralisée, reconstruction automatique, fonctionnalités firmware RAID), il serait nécessaire d’installer un contrôleur RAID compatible avec les disques NVMe.
Les contrôleurs compatibles NVMe RAID pour ce type de serveur incluent généralement :
Dell PERC H755N
Dell PERC H965i
Avant toute décision, il sera également nécessaire de vérifier que le backplane NVMe du serveur supporte le mode RAID.
Actions proposées :
Vérifier la compatibilité du backplane NVMe avec le RAID matériel
Évaluer le besoin réel d’un RAID matériel pour les disques NVMe
Si nécessaire, soumettre une demande de commande pour un contrôleur RAID NVMe compatible
Merci de me confirmer si nous devons procéder à la demande de contrôleur RAID NVMe ou continuer avec la configuration actuelle en LVM RAID1.
